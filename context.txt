Project: Monte Carlo Portfolio Risk Engine — C++ → WebSocket → Python Dashboard

What's been built so far:

A C++ Monte Carlo simulation engine in montecarlo.hpp with a montecarlo.cpp entry point. The engine simulates one-day Value at Risk (VaR) on a two-stock portfolio using Geometric Brownian Motion with correlated asset shocks via Cholesky decomposition. It runs across 10 threads using std::thread, each writing into a separate slice of a shared P&L vector. After all threads join, the full P&L distribution is sorted and VaR (5th percentile), CVaR (mean of worst 5%), and mean P&L are computed and printed.

Current output looks like:

Starting thread 0... (x10)
VaR: -4193.77
CVaR: -5378.06
Mean P&L: 434.411

Tech stack:
- C++17, std::thread, std::mt19937, templates, uWebSockets
- Python: asyncio, websockets, numpy, pandas, matplotlib, seaborn, scipy, yfinance
- Protocol: hand-rolled JSON over WebSocket

Key design decisions already made:
- sigma is not stored on the stock struct — it is derived from sqrt(covariance[i][i])
- Cholesky factor is computed once upfront on the Portfolio object before simulation
- Each thread has its own RNG seeded with std::random_device{}() + threadIndex
- Template parameter T on monteCarloEngine<T> controls scalar precision (use double)
- dt = 1.0/252.0 hardcoded for one trading day horizon

Class structure:

class stock {
public:
    std::string name;
    double stockPrice;
    double mu;
};

public:
    std::vector<stock> stocks;
    std::vector<double> weights;
    std::vector<std::vector<double>> covariance;
    std::vector<std::vector<double>> choleskyFactor;
    double totalValue;
    void computeCholeskyFactor();
    bool verifyCholesky();
    void loadTestCase(); // to be replaced by JSON config reader

struct results {
    std::vector<double> PnL;
    double VaR;
    double CVaR;
    double mean;
};

template <typename T>
class monteCarloEngine {
public:
    results run(Portfolio& portfolio, int simulationNumber);
    void simulateBatch(int start, int end, std::vector<double>& pnlOut, Portfolio& portfolio, int index);
};
